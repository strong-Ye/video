# 🎬 电影清单 Movie Watchlist · 前端演示版

> AI Coding 训练营项目 — **B端管理后台 + C端客户端** 两个独立应用，纯前端 + localStorage（无后端，双击即演示）。
> 视觉风格：Netflix 暗黑影院风（纯黑 #000 + 红 #e50914 + 金 #f5c518）。

---

## 一、如何运行

直接用浏览器**双击**以下两个 HTML 文件即可（需联网以加载 Vue3 CDN）：

| 应用 | 文件 | 说明 |
|------|------|------|
| **C端 客户端** | `index.html` | 用户侧：登录注册、电影浏览、影评、影单分类 |
| **B端 管理后台** | `admin.html` | 运营侧：电影增删、影评删除、用户查看 |

> 两个应用**共享同源 localStorage**，因此：在 C端注册的用户、写的影评，登录 B端后台即可看到并管控。

## 二、演示账号

- **B端管理员**：`admin` / `admin`（已内置，角色为 admin）
- **C端用户**：在 `index.html` 直接「注册」即可（用户名 + 密码），注册时自动生成「假实名」用于 B端展示。

## 四、功能清单

### C端（客户端）
- ✅ **账号体系**：注册 / 登录 / 登出，session 守卫（未登录访问受保护页跳转登录）。
- ✅ **电影浏览**：45 部影片分 **5 页** 长列表（每页 9 部），底部红色高亮翻页栏。
- ✅ **跨页搜索**：搜索覆盖全部 5 页 + 推荐片库，结果区提示「跨 5 页命中 X 部」。
- ✅ **海报三级回退**：本地 `./posters/<tmdbId>.jpg` → 原始 TMDb → 内联 SVG 渐变占位（**永不空白**）。
- ✅ **我的影评**：电影详情页写评分+评论；「我的影评」页查看过往、编辑、删除。
- ✅ **我的影单**：自定义多分类（如「周末必看」「科幻神作」），分类内增删影片。
- ✅ AI 推荐页（基于已有评分/影评）。

### B端（管理后台）
- ✅ **电影管控**：统计电影总数；表单新增电影；按行删除电影（C端同步减少）。
- ✅ **影评管理**：列表全部影评，搜索；删除时弹出确认并展示「上门查水表」玩笑风控话术。
- ✅ **用户管理**：列出注册用户，展示用户名 + **假实名** + 脱敏证件号 + 注册时间。

## 五、数据模型（localStorage 键）

| 键 | 内容 |
|----|------|
| `mw_users` | 用户（含假实名、角色） |
| `mw_movies` | 电影目录（B端增删改此数组） |
| `mw_reviews` | 影评（C端写，B端可删） |
| `mw_watchlists` | 自定义影单分类 |
| `mw_wl` | 状态式清单（想看/在看/已看） |
| `mw_session` | 当前登录态 |

## 六、与技术方案的关系

本演示版为《03_电影清单技术方案_V1.0》的**前端形态**：无后端、数据存浏览器，但业务逻辑、页面结构、数据字段与方案一致；`store.js` 即「数据访问层」的桩实现，未来接入 Spring Boot + MySQL 时，只需将其方法替换为 Axios 调用即可平滑升级为真实全栈。
